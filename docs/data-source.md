# Data sources

`stockheartbeat-mcp` separates "what tools exist" from "where the data comes
from". The `DataSource` interface in [`src/sources/index.ts`](../src/sources/index.ts)
defines the three operations every source must implement:

- `getCurrentHeartbeat(symbol)` -- latest closed bucket
- `getRecentHeartbeats(symbol, limit)` -- short timeline, oldest first
- `getSummary(symbol, window)` -- rolling window summary

This indirection means the MCP tool layer never changes when we swap sources.

## Sources

| Source            | Status      | When to use                                              |
|-------------------|-------------|----------------------------------------------------------|
| `mockSource`      | Shipped     | Offline development, demos, CI. Default for v0.1.        |
| `restSource`      | Shipped (Phase 0) | Reads live data from `mcp-host` HTTP API on VPS/Mac. |
| `binanceSource`   | Reserved (v0.3)   | Will subscribe to Binance public WS and aggregate in-process. |

### `mockSource` (v0.1)

Reads [`fixtures/btcusdt.json`](../fixtures/btcusdt.json), validates it against
the Zod schema, and serves it deterministically. The fixture is synthetic and
generated by [`scripts/build-fixture.mjs`](../scripts/build-fixture.mjs) with a
fixed seed; re-running the script reproduces the file byte-for-byte.

Override the fixture path:

```bash
HEARTBEAT_FIXTURE_PATH=/path/to/your.json npx -y @stockheartbeat/mcp
```

### `restSource` (Phase 0)

Calls the `mcp-host` HTTP API (see `micro` repo `src/mcp-host.mjs`):

```
GET  {HEARTBEAT_API_BASE}/v1/heartbeat/current?symbol=BTCUSDT
GET  {HEARTBEAT_API_BASE}/v1/heartbeat/recent?symbol=BTCUSDT&limit=20
GET  {HEARTBEAT_API_BASE}/v1/summary?symbol=BTCUSDT&window=5m
POST {HEARTBEAT_API_BASE}/v1/summarize  { "symbol": "BTCUSDT", "window": "5m" }
POST {HEARTBEAT_API_BASE}/v1/descriptive/commit   (Phase 1, auth required)
GET  {HEARTBEAT_API_BASE}/v1/descriptive/leaderboard
GET  {HEARTBEAT_API_BASE}/v1/agents/{agent_id}/track_record
GET  {HEARTBEAT_API_BASE}/v1/narrative/latest?symbol=BTCUSDT&window=5m  (public, no key)
```

Phase 1 descriptive tools (`commit_descriptive`, `get_agent_track_record`) call these
endpoints via [`src/lib/heartbeatApi.ts`](../src/lib/heartbeatApi.ts). **MCP never
connects to the descriptive JSONL store directly** — HTTP only.

Selection:

```bash
HEARTBEAT_SOURCE=rest \
HEARTBEAT_API_BASE=https://api.stockheartbeat.com \
HEARTBEAT_API_KEY=your-pilot-key \
npx -y @stockheartbeat/mcp
```

### `binanceSource` (planned)

For self-contained demos without any backend. The MCP process will connect to
a public Binance spot trade stream and aggregate dollar-notional buckets
in-memory using the same threshold semantics as the upstream service.

## Selection precedence

1. `HEARTBEAT_SOURCE=rest` or `mock`
2. If unset and `HEARTBEAT_API_BASE` is set -> `rest`
3. Default -> `mock` (CI and offline demos)

`HEARTBEAT_FIXTURE_PATH` only applies to `mock`.
